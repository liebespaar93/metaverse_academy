property 란?

특성 같은 느낌으로 변수 앞에 선언하는 것으로 꾸밈 효과를 받는다.


```UPROPERTY()``` 이와 같은 함수형을 띄고 있으며

```txt
UPROPERTY([specifier, specifier, ...], [meta(key=value, key=value ...)])
Type VariableName;
```

상위 의 예제 코드처럼 원하는 변슈 위에 선언하는 것으로 만들어 줄 수 있다.\
이러한 프러프티를 설정하여 언리얼에서 blueprint에 제한을 주거나 쓰기 허용을 주는등 다양한 옵션을 설정 할 수 있다.

프로퍼티 지정자로는 나중에 많이 쓰는 것들 위주로 정리하겠다.

| 지정자 | 설명 |
| --- | --- |
| BlueprintReadOnly | 블루프린트에서 읽기만 가능하게 합니다 |
| BlueprintReadWrite | 블루프린트에서 읽기와 쓰기가 가능하게 합니다 |
| EditAnywhere | 어디어세나 수정이 가능합니다 |
| EditDefaultsOnly |  프로퍼티 창에서 편집이 가능합니다. |
| EditInline | 인스펙터 안에서만 수정이 가능합니다 |
| VisibleAnywhere | 어디어세든 확인 가능하지만 수정할 수는 없습니다. |
| VisibleDefaultsOnly | 아키타입에서 확인 가능하지만 수정은 할 수 없습니다. |
| VisibleInstanceOnly | 인스턴스에서만 확인 가능하지만 수정은 할 수 없습니다 |
| Category="TopCategory|SubCategory|..." | 카테고리 이름을 정해준다 |
| meta | 아직 정리가 되지않았다. |

> [!WARNING]
> ```edit``` 과 ```visible```은 등증 하나만 사용하길 권장합니다.
